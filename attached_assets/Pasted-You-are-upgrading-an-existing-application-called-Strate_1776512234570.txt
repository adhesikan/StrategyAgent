PROBABILITY ENGINE
==================================================

Create a modular Probability Engine that calculates a final score from 0 to 100.

Inputs:
- generated setup
- market data
- symbol context
- optional news sentiment
- optional analyst sentiment
- risk structure

Outputs:
- finalScore
- score breakdown
- reasons
- warnings
- grade

Scoring categories and target weights:

1. technicalScore = 30%
2. realtimeScore = 25%
3. newsScore = 15%
4. analystScore = 15%
5. riskScore = 15%

Formula:
finalScore =
  0.30 * technicalScore +
  0.25 * realtimeScore +
  0.15 * newsScore +
  0.15 * analystScore +
  0.15 * riskScore

Grade mapping:
- A+ = 85+
- A = 75–84
- B = 65–74
- C = below 65

Technical score should consider:
- trend alignment
- setup/pattern score
- breakout or pullback quality
- relative strength if available
- structure clarity

Realtime score should consider:
- live volume vs average
- price holding/breaking key level
- intraday trend strength
- liquidity / spread quality
- whether confirmation is improving or failing

News score should consider:
- positive vs negative recent news
- headline velocity
- catalyst presence
- direction alignment with setup

Analyst score should consider:
- normalized analyst rating
- upgrades / downgrades if available
- target upside/downside if available

Risk score should consider:
- reward/risk ratio
- stop clarity
- volatility-adjusted risk
- whether the setup is too extended

IMPORTANT:
The score system must be explainable.
Always generate:
- 3 to 5 “Why this trade is strong” reasons
- 2 to 4 “What to watch / weakness” warnings

INSTRUMENT SELECTION ENGINE
==================================================

Create an engine that recommends the best vehicle for the trade idea.

Input:
- setup direction
- trade horizon
- probability score
- option chain data
- user preferences
- liquidity metrics
- implied volatility context if available

Possible outputs:
- Stock
- Long Call
- Long Put
- Bull Call Debit Spread
- Bear Put Debit Spread

Decision framework:

Recommend Stock when:
- options liquidity is weak
- bid/ask spreads are too wide
- user prefers simplicity
- the move is expected to be steady and not highly leveraged
- options pricing is unattractive

Recommend Long Call when:
- bullish setup
- high conviction
- options liquidity is good
- delta is appropriate
- IV is not excessively expensive
- user allows long premium

Recommend Long Put when:
- bearish setup
- high conviction
- options liquidity is good
- delta is appropriate
- IV is not excessively expensive
- user allows long premium

Recommend Bull Call Debit Spread when:
- bullish setup
- defined risk preferred
- IV is moderate or slightly elevated
- expected move is meaningful but not extreme
- user allows debit spreads

Recommend Bear Put Debit Spread when:
- bearish setup
- defined risk preferred
- IV is moderate or slightly elevated
- expected move is meaningful but not extreme
- user allows debit spreads

Always provide:
- recommended instrument
- alternative instrument
- “why this vehicle fits”
- “what makes it safer / better”
- “what tradeoff it has”

==================================================
OPTION CONTRACT EVALUATION RULES
==================================================

For options, evaluate contracts and spreads using a suitability score.

Single-leg long option preferences:
- delta around 0.50 to 0.70 preferred
- good liquidity
- acceptable open interest
- acceptable volume
- tight bid/ask spread
- expiration aligned with expected trade horizon
- avoid extremely far OTM contracts by default

Debit spread preferences:
- long strike near ATM or slightly ITM
- short strike near expected target area
- defined max loss / max profit shown clearly
- debit reasonable relative to width
- both legs liquid enough

Option suitability scoring should consider:
- bid/ask spread percentage
- open interest
- volume
- delta
- DTE fit
- IV reasonableness
- slippage risk

Add warnings for:
- low open interest
- wide spread
- too little volume
- expiration too close
- excessive theta risk
- earnings/event risk
- overly speculative far OTM contract
